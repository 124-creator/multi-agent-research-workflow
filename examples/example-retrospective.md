## 1. 执行了什么 (What was done)

- 读取原始建模数据（160 行 × 63 列，0 缺失，正负样本 1:3）。
- 按规则删除 26 个字段（ID、经纬度、数据源说明、估算备份等），保留 37 列（36 特征 + 1 目标）。
- 做数据质量检查：缺失值、高共线性（Pearson |r| ≥ 0.9）、正负样本区分度、数值范围。

## 2. 得到了什么结果 (Results)

### 交付文件

| 文件 | 位置 | 说明 |

| 数据质量检查报告.md | step_01/ | 缺失 / 共线性 / 区分度 / 范围 |

### 关键数据 / 指标

- 清洗后形状：160 × 37；高共线性对：6 对；强区分度特征：土壤、降雨、地形、植被。

## 3. 发现了什么问题 (Issues found)

- **高共线性**：坡度 / 起伏度 / 粗糙度三者 r > 0.98 → 线性模型系数不稳 → 特征工程阶段去冗余。
- **类别不平衡**：正负 1:3 → Accuracy 虚高 → 训练阶段用 class_weight，重点看 PR-AUC / Recall。
- **坡向周期性**：0–360° 直接用于线性模型不合理 → 转 sin / cos。

## 4. 做了什么决策 (Decisions)

| 决策 | 原因 | 后续影响 |
|---|---|---|
| 删除经纬度 | 防止空间记忆 | 模型学风险机制而非位置 |
| 删除事件日期 | 负样本日期为配对生成 | 避免时序泄漏 |
| 不删除任何样本 | 样本量本就不多 | 全部保留，靠交叉验证评估 |
