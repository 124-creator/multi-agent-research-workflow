GPT-Pro × Claude × Codex

先把方案审对，
再把执行做对。

ResearchLoop V4.1 把 AI Agent 协作拆成两个可循环闭环：Loop 1 由 GPT-Pro 生成/修复方案，Claude 独立检验并给出修改意见；Loop 2 由 Claude 给出 Plan/Review，Codex 执行、测试并提交复盘，再交给 Claude 终检。

启动双 Loop 演示复制协作提示词
GPT-Pro负责 Loop 1 的方案生成与方案修复，把 Claude 的审查意见转化为更稳的路线。
Claude负责独立检验、提出 BLOCKER/RISK/PASS，也负责 Loop 2 的 Plan 与终检。
Codex负责按 Plan 执行、测试、记录 diff 和复盘报告，把交付证据落到文件与命令中。

Loop 1：方案校验循环

GPT-Pro 产出方案 → Claude 独立审查 → GPT-Pro 根据意见修复 → Claude 再审，直到方案能够进入执行层。

GPT-Pro 方案/修复⇄Claude 独立检验

Loop 2：执行验证循环

Claude 给出执行 Plan → Codex 执行/测试/复盘 → Claude 检查 diff、测试结果与复盘，直到成为 G3 候选。

Claude Plan/Review⇄Codex 执行/复盘
